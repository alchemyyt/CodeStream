Stream Code

Workflow (flujo de trabajo)

Es una secuencia de tareas organizadas de tal manera que tengas mejor eficiencia.

Flujo moderno

El ciclo moderno se habla que todos los departamentos están conectados de tal manera que trabajan uno con otro haciendo un flujo de trabajo.

El principio shift left

Es básicamente no posponer la revisión del proyecto, es más difícil hacer revisiones y correcciones al final.
Y otras de sus ventajas es que tendrás todas las herramientas unificadas: “Todo en uno”, para no tener que saltar entre sí ir al editor, luego ir a GitHub, luego a escribir documentación… que esto es lo que rompe el “flujo”.

Editor de codigo moderno

Hoy en dia los programadores hacen todo en su editor de texto, en este caso voy a usar codestream que tiene GitHub(para llevar un control de versiones online),jira(para gestionar tareas y llevar seguimiento de errores) y slack(para comunicacion entre en equipo)
Enlazo github y en insues con jira en vscode dentro de la extension de codestream

Flujo moderno pasos

  1. Creamos un ticket en el propio codestream
    Creamos una rama
    Notificamos a todos en que vamos a trabajar
  2. Preguntamos Como deveriamos hacer el trabajo
    Recibimos las sugerencias
    Explicamos lo que vamos a hacer
  3. Antes de hacer el pull request
    Preguntamos lo que se nos ocurra en el transcurso del trabajo a un compañero
    Investigamos por nuestra cuenta
  4. Todavia Antes de hacer el pull request
    Volvemos a preguntar
    Se incorpora el pull request
  5. Atentes de que se apruebe el pull request
    Que el jefe te de el visto bueno formalmente
    Asi hacemos menos pasos
  6. Jefe hacer el merge aprobando el pull request
    Al que le toque incluye documentacion
    A otro trabajo
  7. Este flujo de trabajo es 33% mas rapido
    Este flujo es por pasos pero al final ya todo esta hecho, se trabaja juntos
    El otro flujo es la reunion al final y es mas lento
